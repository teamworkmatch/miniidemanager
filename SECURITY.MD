# Security Policy

## Reporting a Vulnerability

Tim pengembang Mini File Manager IDE berkomitmen penuh untuk menangani celah keamanan dengan serius. Kami sangat menghargai upaya Anda dalam mendeteksi dan melaporkan temuan kerentanan secara bertanggung jawab demi menjaga ekosistem open-source yang aman.

Untuk melaporkan masalah atau celah keamanan (*vulnerability report*), harap **TIDAK** membuka laporan celah secara publik melalui GitHub Issues. Kirimkan detail temuan Anda melalui email ke:

📧 **your-security-email[at]domain[dot]com** *(Sesuaikan dengan email Anda)* Dengan menyertakan kata **"SECURITY VULNERABILITY"** pada subjek email.

Tim kami akan mengirimkan respons awal dalam waktu 1x24 jam untuk memverifikasi laporan Anda. Setelah itu, pelapor akan terus mendapatkan informasi berkala mengenai kemajuan proses perbaikan (*fix execution*) hingga pengumuman rilis patch terbaru diterbitkan.

## Disclosure Policy

Ketika tim pengembang menerima laporan celah keamanan yang valid, proses penanganan akan dikoordinasikan melalui langkah-langkah terstruktur berikut:

1. **Konfirmasi & Analisis:** Memverifikasi masalah yang dilaporkan, menentukan tingkat keparahan (*CVSS Score/Impact*), dan mengidentifikasi versi aplikasi yang terdampak.
2. **Audit Kode Komprehensif:** Memeriksa seluruh basis kode untuk memastikan tidak ada variasi celah serupa di modul fungsionalitas lainnya.
3. **Penyusunan Patch:** Membuat kode perbaikan (*hotfix*) sesegera mungkin tanpa mengganggu stabilitas ekosistem inti aplikasi.
4. **Rilis Terkendali:** Menerbitkan versi pembaruan yang aman langsung ke repositori utama GitHub dan menginformasikannya kepada komunitas pengguna.

## Comments on this Policy

Jika Anda memiliki saran, masukan, atau kritik mengenai bagaimana kebijakan keamanan ini dapat ditingkatkan agar menjadi lebih baik, silakan ajukan rekomendasi Anda melalui *Pull Request* (PR).
